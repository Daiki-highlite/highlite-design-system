# Inbound Triage — Ore 02

問い合わせ受領から初回MTG設定までを **24時間以内**に完結させる。

## 入力

- 問い合わせ本文（メール / フォーム / SNSメンション）
- 送信元情報（個人名・会社名・役職・送信元ドメイン）
- 受領チャネル（自社サイト / 紹介 / イベント / SNS）

## アウトプット

1. **3行サマリ**（業界 / 規模 / 想定ニーズ）
2. **初回返信ドラフト**（3パターン）
3. **Notion: Ore Pipeline 新規レコード**
4. **担当アサイン候補**（最大2名）
5. **Push文**（`#ore-リード・アサイン情報-cc` 投稿用）

## AIプロンプト例

### 一次トリアージ

```
以下の問い合わせ文を読み、3行サマリ（業界 / 規模 / 想定ニーズ）を抽出して。
次に初回返信ドラフトを3パターン作って:
- A: 課題深掘り型（30分のヒアリングMTGを提案）
- B: 事例提示型（類似事例リンクを添付しつつヒアリングを提案）
- C: 短期相談型（1時間の有料スポット相談を選択肢として提示）

問い合わせ本文:
[本文]

送信元情報:
[氏名・役職・会社名・ドメイン]

受領チャネル:
[自社サイト / 紹介 / etc.]
```

### 仮説リサーチ

```
[企業名] のWebサイト・直近1年のプレスリリース・採用ページから、
- 主要事業
- 直近の戦略変化
- 想定される課題仮説（3つ）
を抽出して。各仮説に「Highliteが提供できる価値」を一行ずつ添えて。
```

### 担当アサイン候補

```
../../_shared/stakeholder-map.md のOre RACIと、現在のメンバー稼働状況 [リンク] を踏まえ、
本案件の主担当候補を2名挙げて。各候補について:
- 適性スコア（10点満点 / 根拠）
- 想定追加稼働
- 既存案件との相互作用
```

## 初回返信のトーンチェック

`../_shared/brand-voice.md` の **チェック項目** を必ず通す。特に：
- 「ご提案させていただきます」→「提案します」など、過剰敬語を削る
- 顧客の課題に**触れた一文**を必ず入れる（テンプレ感を消す）
- MTG候補日は3つ提示（時間帯のバラエティを持たせる）

## エラー時の挙動

- **24h経過しても返信できない**: 担当者DM → さらに4h経過で `#ore-リード・アサイン情報-cc` に `@here` でエスカレーション
- **送信元が不明（フリーアドレス、無記名）**: 初回返信は最小限（「ご連絡ありがとうございます。〜について教えてください」のみ）
- **既存顧客からのインバウンド**: Crystaフェーズの該当レコードを参照し、関係性を踏まえた返信に切り替える
