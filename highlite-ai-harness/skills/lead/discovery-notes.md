# Discovery Notes — Ore 03 (BANTC)

初回MTGの前後で BANTC（Budget / Authority / Need / Timing / Competitor）を埋め、Disco Note を作る。

## BANTCの定義（Highlite版）

| 軸 | 聞きたいこと | 5段階の目安 |
|---|---|---|
| **B**udget | 予算枠の有無・規模感 | 5: 既に確保済み / 1: 全く未確定 |
| **A**uthority | 意思決定者は誰か・今日の相手は何者か | 5: 決裁者本人 / 1: 担当者の担当者 |
| **N**eed | なぜ今、なぜHighliteか | 5: 明確な痛み + 期限あり / 1: 漠然とした関心 |
| **T**iming | いつまでに何を | 5: 3ヶ月以内に着手必須 / 1: 期限なし |
| **C**ompetitor | 比較対象・既存ベンダー | 5: Highlite一択で進めたい / 1: 数社で本格コンペ |

合計 25点満点。**15点以上で提案ステージへ昇格、8点未満でCrystaモニタリング行き**。

## MTG前の準備プロンプト

```
[企業名] の公開情報（HP / プレス / 採用ページ / IR / 経営者の発信）から、
BANTCの仮説を埋めて。各軸2行以内、根拠リンクをつけて。
わからない軸は「不明 / MTG中に必ず聞く」と明記。
```

## MTG中の質問テンプレ

`../_shared/brand-voice.md` のトーンを保ちつつ、以下を**自然な会話の流れ**で埋める：

- B: 「今期、この領域にどのくらいリソース割かれている想定でしょうか？」
- A: 「最終的に意思決定されるのはどなたの想定でしょうか？」
- N: 「これが3ヶ月後・半年後に解決されていない場合、何が困りますか？」
- T: 「逆に、いつ頃までに動き出したいというイメージはありますか？」
- C: 「他に検討されている方法・パートナーがあれば教えてください」

**避けるべき**: BANTCを直接聞く（「予算は？」「決裁権ありますか？」）。会話の流れを切らずに、相手の言葉から拾う。

## MTG後の処理

### 1. 文字起こし → BANTCスコアリング

```
議事録 [本文 or Zoom文字起こしリンク] からB/A/N/T/Cの言及箇所を抽出し、
各5段階で評価して。各評価に「該当する発言の引用」を必ず添える。
判断つかない箇所は「保留」とし、追加ヒアリングが必要なら質問案も併記。
```

### 2. Disco Note作成

Markdownで以下の構成：

```
# Disco Note: [企業名] / [日付]

## 出席者
- 先方: [氏名・役職]
- Highlite: [氏名]

## 議論サマリ（5行以内）
...

## BANTCスコアリング
| 軸 | スコア | 根拠（引用） |
| B  |  X/5  | 「...」 |
| A  |  X/5  | 「...」 |
| N  |  X/5  | 「...」 |
| T  |  X/5  | 「...」 |
| C  |  X/5  | 「...」 |
| **計** | **XX/25** | |

## 次アクション
- [ ] 期日 / 担当 / 内容

## 提案イメージ仮説（3行）
...
```

### 3. ステージ判定

```
BANTCスコア合計 [数値] と Disco Note [本文] を踏まえ、
次の打ち手を以下から選んで判定理由つきで提示:
- A: 即提案（Ore 04へ）
- B: 追加ヒアリングMTG設定（Ore 03 を1回繰り返す）
- C: ナーチャリング行き（Crysta索敵対象として登録）
- D: 一旦離脱（理由を明記、3ヶ月後にCrysta索敵で再評価）
```

スコア15以上の場合は冒頭に `@here :rocket: 提案ステージ昇格` を追加。

## 注意

- BANTCは**判定の道具**であって、人間の関係性を測るものではない。スコア以前に「この人と仕事したいか」も率直に書き残す
- スコア低 = 価値なし、ではない。Crystaに送って関係性を維持する
- 数値だけで決めない。Disco Note の **議論サマリ** と **提案イメージ仮説** が判断の主役
