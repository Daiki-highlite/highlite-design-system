# Notes — マーケティング議事録管理

会議・議論の内容を `marketing/mom.md` に蓄積する。過去の記録は歴史として保持し、新しい議論のみ追記する。

## このスキルが扱う範囲

| 役割 | 対象ファイル | トリガーフレーズ |
|---|---|---|
| 議事録の追記 | `skills/notes/marketing_notesrule.md` | 「議事録に追記して」「まとめて」「記録して」「mom.mdに書いて」 |

## 共通の作法

1. **必ず参照する**: `../../CLAUDE.md` / `../_shared/glossary.md`
2. **既存の内容は読むだけ** — 編集・削除・移動は絶対にしない
3. **追記は末尾のみ** — 既存セクションの間には挿入しない
4. **日付は今日の実際の日付を使う**

## 手順

### 1. 既存ファイルを読む

`skills/notes/marketing_notesrule.md` を読み、現在のセクション構成を確認する。

### 2. 今回の会話を整理する

以下の3点を会話から抽出する：

- **決定事項**：「こうする」と明示的に決まったこと（曖昧なものは含めない）
- **ネクストアクション**：誰が・何を・いつまでにやるか
- **未決事項**：まだ決まっていない・確認が必要なこと

### 3. セクションを末尾に追記する

```
---

## YYYY年MM月DD日 | [トピック名]

### 決定事項
- ...

### ネクストアクション
- [ ] ...

### 未決事項
- [ ] ...
```

トピック名は会話の内容を一言で表すもの（例：「LP評価・ターゲット深掘り」「DL資料の構成確定」）。

### 4. 追記後に確認

追加したセクションの内容を簡潔にユーザーに伝える。

## 設計思想

- **記録は判断の積み上げ** — 何を決めたかより、なぜ決めたかを残す
- **過去は変えない** — 書き直すより、新しい観点として追記する
- **シンプルに保つ** — 書くことを増やすより、必要なことだけを簡潔に
