# Onboarding — 新メンバー / パートナーの受け入れ

最初の1〜4週間で、Highliteの**フェーズ言語と動き方**に馴染んでもらう。詰め込みすぎない、薄めすぎない。

## オンボーディングの種類

| 種別 | 対応 | 期間 |
|---|---|---|
| 新規社員 | フル版 | 4週 |
| 業務委託パートナー（継続） | 中版 | 2週（PJ開始前1週 + 開始後1週） |
| 業務委託パートナー（スポット） | 軽版 | 開始前2-3日 |

## フル版（新規社員 / 4週）

### Week 1: 文脈を浴びる

```
Day 1
- DHから「Highliteとは / 余白の哲学」（30分）
- ../README.md / ../CLAUDE.md / ../_shared/brand-voice.md を読む（自学）
- ../_shared/glossary.md を読む（社内用語の習得）
- ai-dashboard.html を眺める（フェーズ感の獲得）

Day 2-3
- ../_shared/stakeholder-map.md と現メンバーとの自己紹介MTG（各15分）
- 過去の事例記事 ../internal-marketing/case-study.md 5本を読む

Day 4-5
- 過去のキックオフ資料3件、Disco Note 5件を読む
- 「Highliteらしいと感じたこと / ピンと来なかったこと」を書き出す
```

### Week 2: 観察と質問

```
- 既存PJの定例MTGに2-3本オブザーバー参加
- パイプラインレビューに同席（../biz-mgmt/pipeline-review.md）
- 1on1 #1 を実施（../people-mgmt/1on1-prep.md）
- Will/Can/Must を初版で言語化
```

### Week 3: 小さく動く

```
- 担当領域を1つアサイン（フェーズの一部 or 1PJのサポート）
- 最初のアウトプット = ../_shared/brand-voice.md レビュー必須
- パイプラインレビューで観察した「気になる案件」1つを発表
```

### Week 4: 一人で回す

```
- 担当領域で1つの完結したアウトプット
- 1on1 #2 — 4週間の振り返り
- Will/Can/Must の更新
- ../_shared/stakeholder-map.md と ダッシュボード Member セクションを更新
```

## 中版（継続パートナー / 2週）

### 開始前1週

```
- ../README.md / ../CLAUDE.md / ../_shared/brand-voice.md を共有
- ../_shared/glossary.md でフェーズ言語を確認
- 該当PJのキックオフ資料・Disco Note を共有
- DHまたはDelivery Leadと顔合わせ（30分）
```

### 開始後1週

```
- 最初の納品物に対して、Delivery Leadから細やかなフィードバック
- 1on1 #1 を実施（短縮版 20分）
- ../_shared/stakeholder-map.md パートナー一覧に追記
```

## 軽版（スポット / 2-3日）

```
- ../_shared/brand-voice.md と該当PJ概要のみ共有
- スコープと検収基準の確認MTG（30分）
- 納品 → 検収 → 請求は ../biz-mgmt/financial-snapshot.md
- 必要なら ../_shared/stakeholder-map.md に「スポット」表記で追記
```

## AIプロンプト例

### オンボーディングカリキュラム生成

```
新メンバー [氏名 / 役割 / 経歴] のオンボーディングカリキュラムを、
上記フル版テンプレを基に個別カスタマイズして:

- 経歴から既知の領域 → 学習時間を短縮
- 未知の領域 → 重点配分
- Will（事前に聞き取り済みなら）→ 最初のアサイン候補と接続
- 1on1の問いも事前に3つ準備
```

### 既存ナレッジへの導線

```
[新メンバーの興味関心 / 経歴] を踏まえ、
オンボーディング期間中に読むべき社内ドキュメントを優先順位付け:

- 必読（Day 1〜3）
- 推奨（Week 1）
- 余裕があれば（Week 2-4）

各ドキュメントに「読むべき理由」を1行ずつ。
```

### Will/Can/Must 初版言語化サポート

```
[新メンバー氏名] との1on1メモ [リンク] から、
Will/Can/Must の初版ドラフトを起案:

- Will: 本人が「これをやりたい」と話した内容
- Can: 経歴 + 本人が「これはできる」と話した内容
- Must: Highlite側から期待していること（事前にDHと擦り合わせ済みのもの）

各2-3行。本人の言葉を尊重し、形容詞を盛らない。
```

## 注意

- **詰め込みすぎない** — 4週で全部教えようとしない。半年スパンで馴染んでもらう前提
- **薄めすぎない** — 「自由にやって」だけは放任。必ずメンターと1on1のペースを設定
- **パートナーも対等に扱う** — 業務委託だからオンボーディング省略、はやらない
- **Welcome文化を残す** — 新メンバー時の歓迎は、既存メンバーにも組織の温度を思い出させる
- **離脱時のオフボーディング** — 入る作法と同じくらい、出る作法も大事（別ドキュメントで管理推奨）
