# Performance Review — 半期 / 通期レビュー

評価のためというより、**Will / Can / Must の現在地を一緒に確認する場**。点数化より対話。

## レビューの目的

- メンバーの **Will / Can / Must の更新**
- 過去半年のハイライトと、磨きが必要な領域の共通認識
- 次半期の役割と、報酬・働き方の調整判断
- 組織として「何を価値とするか」の言語化を全員で重ねる

**目的にしないもの**: 順位付け、相対評価、過去の失敗の追及

## サイクル

| 時期 | 何をするか |
|---|---|
| 半期末（年2回） | 半期レビュー。Will/Can/Mustの更新 + 役割調整 |
| 年度末 | 通期レビュー（半期の延長） + 報酬見直し |
| 四半期 | 軽い棚卸し（1on1の中で扱う、独立MTGなし） |

## 入力

- 過去6ヶ月の1on1メモ
- 過去6ヶ月に参加したPJ一覧（役割含む）
- 過去6ヶ月のSlack発言（公開チャンネル + アサインPJスレッド）
- 過去6ヶ月のアウトプット例（資料 / 議事録 / デザイン etc.）
- 顧客からのフィードバック（あれば / 検収時コメント等）
- 現Will/Can/Must記述

## AIプロンプト例

### レビュー資料の素材集約

```
[氏名] の過去6ヶ月のレビュー素材を集約:

入力:
- 1on1メモ: [リンク x6]
- 参加PJ: [リスト]
- Slack発言（直近180日 / 公開チャンネル）: [リンク]
- アウトプット例: [リンク x N]
- 顧客フィードバック: [リンク]

抽出して欲しいもの:
- ハイライトとなった場面（3-5）
- Willの記述変化（半期前と今）
- Can（伸びた / 新しく身についた / これから伸ばしたい）の素材
- Mustに対する果たし方の素材
- 摩擦・違和感の素材（過剰解釈せず、本人発言から）

出力は上記レビュー資料のフォーマットに沿って、ドラフトとして提示。
```

### 自己振り返りシート（メンバー側用）

```
[氏名] が自分で記入する自己振り返りシートのテンプレを起案:

セクション:
1. この半年でやって良かったこと（自由記述）
2. 自分のWillの今の感じ方
3. 新しく「できる」ようになったこと
4. もっと磨きたいCan
5. Highliteに対するリクエスト・提案
6. 来期の自分への問い

全項目「書きたいだけ書いてOK / 書かなくてもOK」と明記。
DHが書いたドラフトと突き合わせる前提で、本人の言葉を尊重する。
```

### レビューMTGアジェンダ

```
[氏名] のレビューMTG（90分）のアジェンダを起案:

- 0-10分: 雑談 + 半期で印象的だった場面のシェア（お互い）
- 10-30分: 自己振り返りシートを本人が話す
- 30-50分: DHからのドラフトと突き合わせ（差分について対話）
- 50-70分: 来期のWill/Can/Must調整 + 役割の方向性
- 70-85分: 報酬・契約条件の話（ある場合）
- 85-90分: 次までに持ち帰ること

「相手が話す時間 > DHが話す時間」を必ず守る。
```

## レビュー後の処理

- レビュー資料 → 本人とDH限定アクセスで保管
- Will/Can/Must更新 → `../_shared/stakeholder-map.md` + `ai-dashboard.html` Member セクション
- 報酬変更 → 契約書改訂 + `../biz-mgmt/financial-snapshot.md` の人件費予測に反映

## 注意

- **書面評価より対話の質** — レビュー資料は、対話を深めるための補助
- **数値評価をしない** — 「5段階で何点」をやらない。形容詞と場面で語る
- **比較しない** — 他メンバーとの相対評価は禁止
- **本人未確認の記述を残さない** — 周囲からの観察は、本人と擦り合わせ済みのものだけ記述
- **同じことを毎回書かない** — テンプレ運用に飽きが出たら、フォーマットを変える勇気を持つ

## レビューの「失敗」パターン

- DHが話しすぎて、本人が黙ってしまう → 進行を逆転させる
- 「できなかったこと」が話の中心になる → ハイライトに戻る
- Will/Can/Must が前回と全く同じ → 半年経って動いてないなら、組織側の責任を疑う
- 報酬の話で温度が下がる → 別途独立した時間で扱う選択肢を持つ
