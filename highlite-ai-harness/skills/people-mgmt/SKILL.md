# People Management — メンバーと向き合う

Highliteは小規模だからこそ、**ひとりひとりの Will / Can / Must が事業の解像度**になる。

## このスキルが扱う範囲

| 役割 | サブスキル | トリガーフレーズ |
|---|---|---|
| 1on1準備 | `1on1-prep.md` | 「[名前]との1on1準備」「振り返り材料」 |
| オンボーディング | `onboarding.md` | 「新メンバー受け入れ」「パートナー初稼働」 |
| 半期 / 通期レビュー | `performance-review.md` | 「評価まとめて」「半期振り返り」 |

## 共通の作法

1. **必ず参照する**: `../../CLAUDE.md` / `../_shared/brand-voice.md` / `../_shared/stakeholder-map.md`
2. **Push先**: 個別の通知は基本DM。チャンネル投稿は最小限
3. **個人情報の扱い**: 評価・1on1メモは限定アクセス。AIに渡す際もコンテキストに残さない指示を添える
4. **WCMフレーム**: Will / Can / Must を全運用の共通言語として使う

## Will / Can / Must の定義（Highlite版）

| 軸 | 問い | 更新頻度 |
|---|---|---|
| **Will**（やりたい） | 仕事を通じて何を実現したいか | 半期 |
| **Can**（できる） | 何ができるか / 何が得意か | 四半期（PJ完了時） |
| **Must**（果たすべき） | この組織で何を果たすことが期待されるか | 月次（PJアサイン時） |

3つの**重なり**が大きいほどパフォーマンスとエンゲージメントが両立する。重ならない時、それは異常ではなく**対話の起点**。

## 設計思想

- **管理ではなく伴走** — 監視ではなく、相手の磨きを助ける
- **評価より対話** — スコアより、一緒に意味を見つける時間を優先
- **個別性を尊重** — 一律のプロセスを押し付けない
- **ダッシュボードのMemberセクションと同期** — Will/Can/Mustの更新は必ずダッシュボード反映
