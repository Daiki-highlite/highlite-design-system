# Social Post — SNS / コンテンツ発信

Ore 01「接点創出」の燃料供給。X / LinkedIn / note を中心に、**月数本**の高密度発信を回す。

## チャネル別の役割

| チャネル | 主な目的 | 想定読者 | 投稿頻度の目安 |
|---|---|---|---|
| **X** | 短い気づき・問い・引用 | 同業・パートナー候補 | 週2-3 |
| **LinkedIn** | 事業判断の語り | 経営層・事業責任者 | 週1 |
| **note** | 思考のプロセス共有 | 共感層・採用候補 | 月1-2 |

## 共通の作法

1. **必ず参照する**: `../_shared/brand-voice.md`
2. **使う言葉 / 使わない言葉**: `../_shared/glossary.md` の対比表
3. **「磨く」という動詞を主役に** — Highliteらしさが立ち上がる
4. **問いで終わる** — 答えで閉じるより、読者の思考を促す

## ネタ源

```
- 事例記事（../internal-marketing/case-study.md）からの引用
- ブランド索敵レポート（Crysta 03）で見つけた業界の動き
- メンバーの1on1（people-mgmt/1on1-prep.md）で出てきた言葉
- 提案書ドラフト（lead/proposal-draft.md）の中で使った比喩
- デザインレビュー（delivery/creative-review.md）で出た判断
```

## AIプロンプト例

### X（短文・気づき）

```
[ネタの素材] を、Xの投稿（140字以内）に変換。
- 主語をHighliteにしない
- 形容詞を盛らない
- 「磨く / 結ぶ / 整える / 余白」など Highlite語彙を1つ含める
- 末尾は問い or 余韻で
- ハッシュタグは使わない（必要時1つだけ）

3案出して。
```

### LinkedIn（事業判断の語り）

```
[業界トレンド or 事例] を、LinkedIn投稿（300字程度）に展開。
構造:
- 1段落目: 観察した事実（具体的に）
- 2段落目: そこから読み取れる「磨きの余地」
- 3段落目: 経営判断としての問い

../_shared/brand-voice.md のトーン適合度をセルフチェックして、3案目を最も「らしい」案にして。
```

### note（プロセス共有）

```
[ネタの素材] から、noteの記事構成を起案（1500-2500字想定）:
- タイトル案 3つ（クリック狙わない / 内容を表す）
- リード（150字）— 何の話で、なぜ書くか
- 本文構成（章立て + 各章の狙い1行）
- 結び — 答えを出さず、読者の判断に委ねる

書き始める前にDH（or Delivery Lead）にDMで「この方向で」確認を取る前提で起案。
```

### 月次運用

```
過去30日のSNS反響データ [リンク] と、Crysta索敵レポート [リンク] から、
来月の投稿計画を起案:
- 各チャネル別: テーマ × 本数
- 連動キャンペーン（同テーマで X → LinkedIn → note と展開）
- やめるトピック（反応が薄い領域）

詳細プランは別途。本リクエストは骨格のみ。
```

## 発信の作法（Highlite流）

1. **量を追わない** — 月数本で質を保つ
2. **新規性より誠実さ** — 既出の話題でも、自分の言葉で語り直せばよい
3. **「役立つ」を狙わない** — 結果として誰かの判断が楽になる、を目指す
4. **競合を名指ししない** — 比較で勝とうとしない
5. **数字でメンバーを評価しない** — エンゲージメントは観察対象であって、評価軸ではない

## 注意

- **公開前のセルフチェック**: `../_shared/brand-voice.md` のチェック項目を必ず通す
- **顧客名を含む投稿**: 必ず Crysta 01 の許諾範囲内であることを確認
- **メンバー名を含む投稿**: 本人にDMで確認、固有名表記より「あるメンバー」表記を優先
- **数字を使う場合**: 出典・期間・比較対象を明記。盛らない
- **AI生成感を消す** — Claude出力をそのまま投稿しない。一度「自分の声」で書き直す
