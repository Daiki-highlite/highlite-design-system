# Pipeline Review — 週次パイプラインレビュー

Oreフェーズ全体の温度感を週次で測り、**判断が必要な案件を炙り出す**。

## 入力

- Notion「Ore Pipeline」全レコード
- 各案件のBANTCスコア / Disco Note
- 直近1週間の活動ログ（メール送付 / MTG実施 / 提案送付）

## アウトプット

```
# Ore Pipeline Review — YYYY/MM/DD

## サマリ
- アクティブリード: X件
- 加重パイプライン額: ¥X.XM
- 平均BANTCスコア: XX.X / 25
- 提案中: X件 / 受注待ち: X件

## ステージ別

### Prospect（BANTC < 8）
- 件数: X
- 動きが鈍いもの: [リスト]

### Discovery（BANTC 8〜14）
- 件数: X
- ステージ滞留が2週超: [リスト]

### Proposing（BANTC 15+ / 提案前）
- 件数: X
- 期日近接: [リスト]

### Negotiating（提案送付済 / 返答待ち）
- 件数: X
- 7日無音: [リスト]

## 判断が必要な案件
- [企業名 or 伏字]: [理由 + 推奨アクション]
- ...

## ステージ移行（先週との差分）
- 昇格: X件
- 降格 / Crysta行き: X件
- 受注: X件
- 失注: X件
```

## AIプロンプト例

### 全件レビュー

```
Notion「Ore Pipeline」のCSVエクスポート [リンク] と
各案件の BANTC + 直近活動ログ [リンク] を読み、
上記フォーマットで Pipeline Review を生成して。

特に：
- 「動きが鈍いもの」の判定: 最終アクションから14日以上経過
- 「ステージ滞留が2週超」: 同じステージで2週以上停滞
- 「7日無音」: 提案送付から7日以上、顧客側からの返信なし
- 「判断が必要な案件」: 担当者では判断できない / Goサインが必要なもの
```

### 加重パイプライン算出

```
各案件の [見積金額 × BANTCスコア / 25] を合算して、加重パイプライン額を算出。
ただし以下は除外:
- BANTC < 8（蓋然性低すぎ）
- ステージ滞留 4週超（実質死に案件）

算出根拠（除外案件リスト含む）も併記。
```

### ステージ別深掘り

```
Negotiating ステージの案件 [リスト] について、
- 提案送付からの経過日数
- 直近の顧客側アクション（メール開封 / 質問 / 沈黙）
- 推奨される次アクション（フォロー / 価格再検討 / Crysta行き判定）
を表形式で出力。
```

## レビューMTG（週次）

毎週月曜の朝、DH + Delivery Lead + BizDev で15分。

進行:
1. 5分: 数字を眺める（驚きの有無を共有）
2. 5分: 「判断が必要な案件」を1件ずつ捌く
3. 5分: 来週やめるべきこと / 続けるべきことを1つずつ決める

## 注意

- **顧客名は伏字**で全社共有 — 実名はDM or 限定された場で
- **加重金額の精度を上げない** — 「だいたい当たっている」レベルで判断材料として十分
- **失注を悪く見ない** — 失注理由のパターン化は、提案の質を上げる材料
- **ステージ移行が起きないPJ** が一番危険 — 動いていないことに気づくのがレビューの主目的
