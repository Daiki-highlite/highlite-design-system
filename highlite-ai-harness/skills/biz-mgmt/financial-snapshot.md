# Financial Snapshot — 月次の財務スナップショット

請求・入金管理と、月次の事業数字を**判断のための形**にまとめる。

## 扱う数字

### 月次サマリ
- 受注額（新規受注の合計）
- 売上計上額（検収完了の合計）
- 入金額（実際に着金した合計）
- 未収残（請求済み × 未入金）
- アクティブPJ数 / 加重Oreパイプライン
- メンバー稼働率（社員 / パートナー別）

### キャッシュ管理
- 月初残高
- 月内入金見込み（請求済み + 入金期日内）
- 月内支出見込み（人件費 + 委託費 + 固定費）
- 月末残高見込み
- ランウェイ（月数）

## 入力

- 銀行口座CSV
- 請求書発行履歴（Notion or 会計ソフト）
- 契約書一覧（受注額の根拠）
- メンバー稼働ログ（タイムシート or 推計）

## 請求・入金フロー

### 請求書発行（Lite検収完了 → 請求）

```
検収報告 [リンク] と契約書 [リンク] から、
請求書記載項目を生成して:
- 件名
- 金額（税込・税抜）
- 支払期日（契約条件 + 月末締め翌月末払い等のルール）
- 振込先
- 備考（PJ期間 / 担当者）
```

### 入金リマインド

```
入金期日が今週の請求 [リスト] について、
顧客向け事前リマインドメール（簡潔・フレンドリー）を起案。
「お忙しいところ恐縮ですが、本日 [X月X日] が支払期日となっておりまして〜」程度。
```

### 入金催促（期日超過）

```
入金期日を [X日] 過ぎた請求 [リスト] について、
催促メールの3段階ドラフト:
- 1次（期日 +1日 / 丁寧・確認トーン）
- 2次（期日 +7日 / 確認 + 経理担当者宛）
- 3次（期日 +14日 / 経営層宛・期限明示）
```

### 入金確認 → Crysta移行

```
通帳CSV [データ] と未収請求リスト [データ] を照合し、
入金マッチング結果を「請求番号 / 入金日 / 差額」の表で出力。
入金完了したPJについて、Crysta移行通知の社内アナウンス文を作って。
```

## 注意

- **金額は伏字運用** — 全社共有では「数百万円規模」「億単位」など `glossary.md` のレンジ表現
- **キャッシュ判断は早めに** — ランウェイ6ヶ月は最終警告ではなく「動き始める」サイン
- **稼働率の精度は粗くてよい** — 完璧な計測より、傾向把握
- **未収残はPJ単位で管理** — 単純合計より、どの顧客が滞っているかが大事
- **会計ソフト連携を急がない** — 月次手作業から始め、規模に応じて自動化を判断

## 季節性

- 年度末（3月 / 12月）: 駆け込み受注で稼働ピーク → リスクログに事前登録
- 四半期末: 顧客側の予算消化案件が増える → Pipeline Review でフラグ管理
