# Kickoff — Lite 01

契約締結から **3営業日以内** にキックオフMTGを実施し、磨きの方向を確定させる。

## 準備物

- 提案書ファイナル版（リンク）
- 契約書（リンク）
- Disco Note（リンク）
- アサインメンバー一覧（`../_shared/stakeholder-map.md` 参照）

## キックオフ資料の構成（標準スライド10枚）

```
01. 表紙: PJ名 / 顧客名 / 開始日
02. なぜ今、これを始めるのか（顧客の言葉から）
03. PJのゴール（測定可能な形で）
04. スコープ（やること / やらないこと）
05. マイルストーン（初月の3つを必ず明示）
06. 体制（Highlite側 / 顧客側 / 役割）
07. コミュニケーション設計（定例 / SLA / 緊急連絡）
08. 検収基準（何をもって完了とするか）
09. 想定リスクと対応（3つ程度）
10. 次の一歩（今日決めること / 今週やること）
```

## AIプロンプト例

### 資料起案

```
提案書 [リンク] と契約書 [リンク] とDisco Note [リンク] から、
キックオフ資料（上記10枚構成）をMarkdownで起案して。

特に：
- スライド02「なぜ今これを始めるか」は、Disco Note の顧客発言から3つ引用
- スライド03「ゴール」は、定性目標 + 必ず1つ以上の定量指標
- スライド05「初月マイルストーン」は、検収可能な粒度で
- スライド08「検収基準」は、「主観的にOK」は禁止。第三者が見ても判定できる形に
```

### マイルストーン設計

```
スコープ [本文] と契約期間 [Xヶ月] から、
マイルストーンを以下構造で提案:

| # | 期日 | 成果物 | 検収基準 | 主担当 |

特に初月は3つ以上、最終月は1つに絞る（Liteの常套）。
```

### 期待値ズレ予測

```
キックオフ資料 [リンク] を読み、
お互いの「期待値ズレが起きそうなポイント」を5つリストアップ。
各ポイントに「キックオフ中に確認しておくべき質問」を1行ずつ。
```

### キックオフ議事録

```
キックオフ録画の文字起こし [リンク] から、
- 決定事項（Decision）
- 持ち帰り（Action）
- 議論で出た懸念（Risk）
の3区分で議事録を作成。
顧客側に共有可能なフォーマット（敬語・伏字なし）で。
```

## キックオフ当日の進行

1. **冒頭5分**: お互いの自己紹介（仕事だけでなく、ひとりの人として）
2. **次10分**: 「なぜ今、これを始めるか」を顧客自身の言葉で語ってもらう
3. **次30分**: 用意した10枚スライドを進行（スライド5〜8で必ず質疑）
4. **次10分**: 想定リスクと、お互いの「困りごと」を率直に共有
5. **最後5分**: 次の一歩を明文化

**避けるべき**: 一方的な説明会。半分は顧客が話している状態を目指す。

## 注意

- キックオフは「決めること」より「**お互いを知る場**」。資料は骨組みで、対話が主役
- スコープ外要望が出たら、その場で「Phase 2 候補」として書き留める（断らないが、安易に飲まない）
- 顧客側の決裁者がキックオフに同席しない場合は、後日キャッチアップMTGを必ず別途設定
