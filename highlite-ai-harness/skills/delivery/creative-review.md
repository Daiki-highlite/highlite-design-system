# Creative Review — Lite / ブランドデザイン全工程

ダッシュボード Flow の **LITE / ブランドデザイン レーン**（10ステップ）を対象に、デザイン中間物とアウトプットをレビュー・整える。

## 対象範囲

```
01. コア価値の特定          ← 言語ワーク
02. ブランド定義            ← 言語ワーク
03. コミュニケーションデザインの設定  ← 言語ワーク
04. テイストの決定          ← ビジュアル準備
05. 要素構成の決定（Claude Design）  ← AI併用ビジュアル
06. デザインシステムの参照（Claude Design）
07. デザイン案の提示（Claude Design）
08. Figmaへの連携
09. Figmaでの細部表現
10. エンジニアまたは先方への展開    ← `handoff.md` へ引き継ぎ
```

## レビューの3つの目線

| 目線 | 何を見るか | 主な質問 |
|---|---|---|
| **Brand Fit** | コア価値・ブランド定義との整合 | これは「余白」を体現しているか |
| **Craft** | 形・色・余白・タイポの完成度 | 細部が雑になっていないか |
| **Feasibility** | 実装・運用可能性 | エンジニア / 顧客側で再生産できるか |

## 各ステップのAIプロンプト例

### Step 01-03: 言語ワーク

```
[コア価値仮説 / ブランド定義 / コミュニケーション設計] のドラフト [リンク] について、
- ../_shared/brand-voice.md のトーン適合度（10点満点 + 根拠）
- 抽象度のバランス（抽象→具体→抽象 になっているか）
- 競合との差別化が生まれているか
を3列で評価して。
直すべき箇所はBefore→Afterで提示。
```

### Step 04: テイスト

```
ムードボード [リンク or 画像説明] を読み解き、
- 想起されるキーワード（5つ）
- 顧客が反応しやすい1点
- ブランド定義 [リンク] とのズレが生じている箇所
を抽出。クライアントへの説明スクリプトも一段落で。
```

### Step 05-07: Claude Design 出力レビュー

```
Claude Design 出力 [リンク] をブランド定義 [リンク] と照合し、
- 忠実度（10点 + 根拠）
- 新規性（10点 + 根拠）
- 実現可能性（10点 + 根拠）
を採点。3軸の合計上位案について、
クライアント提案ナラティブ（採用意図・想起される印象・採用時のリスク）を起案。
```

### Step 08-09: Figma整理 / 細部

```
Figmaファイル [リンク] のレイヤー命名・コンポーネント構造を、
以下チェックリストで監査:

- [ ] Frame名がBEM風 or 機能名で統一されているか
- [ ] AutoLayoutが適切に使われているか（手動位置指定が残っていないか）
- [ ] Variants が状態（default/hover/active/disabled/error）を網羅
- [ ] カラー・タイポ・スペーシングが Variables 経由になっているか
- [ ] レスポンシブ（mobile 375 / tablet 768 / desktop 1280）の各フレーム整備
- [ ] エッジケース（空状態 / エラー / ローディング）の表現

問題箇所は「ファイル内のセクション名 / 修正方針」の表で出力。
```

## レビューの作法（Highlite流）

1. **指摘より問いを立てる** — 「これ違う」より「なぜこの選択？」
2. **Before/After で提示** — 言葉だけのフィードバックは残さない
3. **クライアントの口を借りない** — 「クライアントが嫌がるかも」より、Highliteとして「これは違う」と言う
4. **磨きを止めない** — 60点で止めずに、80点まで持っていく粘り

## デザイン提案時のテンプレ

クライアントへの提案時は、各案について以下3点を必ず添える:

```
■ [案A: 名称]

意図:
[この案がブランド定義のどこに応えているか / 1段落]

想起される印象:
[5キーワード]

採用時のリスク:
[2点・正直に書く]
```

## 注意

- **Claude Design は道具、判断は人**。出力をそのまま採用しない。必ず Brand Fit / Craft / Feasibility を通す
- 顧客レビュー前に、必ず社内レビュー1回（Delivery Lead + DH）
- フォント・画像の **ライセンス確認**を Step 06 のチェックリストに含める
- Figma → 実装の引き継ぎは `handoff.md` に従う
