# Handoff — Lite 04 / ブランドデザイン 10

デザイン → 実装、または Highlite → 顧客側内製チームへの引き渡し。**ここで雑になると、磨いた価値が落ちる**。

## 引き渡す相手

| 相手 | 状況 | 主な引き渡し物 |
|---|---|---|
| 顧客側のエンジニア | 内製実装 | Figma Dev Mode + 仕様書 + 質問対応窓口 |
| 顧客側のデザイナー | 自走化 | Figmaライブラリ + 運用ガイドライン + メンテナンスTips |
| Highliteの委託エンジニア | 実装委託 | 同上 + 内部Slackルート |
| 顧客の運用担当 | 制作物の納品 | 最終データ + 取扱説明 + 改変ルール |

## 引き渡し前のチェックリスト

### 共通

- [ ] 検収基準を顧客と再確認（キックオフ時の定義と齟齬がないか）
- [ ] 納品物のバージョン管理がされているか
- [ ] フォント・画像・素材のライセンス資料が揃っているか
- [ ] 引き渡し後の質問対応窓口・期間が合意されているか

### Figma → 実装

- [ ] Dev Mode が有効
- [ ] Variables（color / typography / spacing / radius）が定義済み
- [ ] コンポーネント Variants が網羅（default/hover/active/disabled/error）
- [ ] レスポンシブのブレイクポイント別 Frame が揃っている
- [ ] エッジケース（empty / error / loading）の表現あり
- [ ] アクセシビリティ要件（コントラスト / フォーカス / aria）の注釈

### ブランドガイドライン → 顧客自走

- [ ] ブランドピラー / バリュー / トーンの定義が独立した1ファイルに集約
- [ ] 「やってよいこと / やってはいけないこと」の対比表
- [ ] ロゴ使用ルール（最小サイズ / クリアスペース / 不可用例）
- [ ] テンプレート（プレゼン / SNS / メールヘッダー など）

## AIプロンプト例

### 実装ガイド起案

```
Figma Dev Mode の出力 [リンク] と デザインシステム定義 [リンク] から、
エンジニア向け実装ガイド（Markdown）を起案して。
構成:

1. 概要（実装範囲 / 技術スタック前提）
2. デザイントークン一覧（カラー / タイポ / スペーシング / シャドウ）
3. コンポーネント一覧（各コンポーネント: Props / States / a11y要件）
4. レイアウト規則（グリッド / レスポンシブブレイクポイント）
5. アニメーション定義（duration / easing / 適用箇所）
6. アクセシビリティ要件（WCAG準拠レベル / ARIA注意点）
7. よくある実装ミス（事前共有）
8. 質問窓口とSLA
```

### 顧客自走化ガイド起案

```
ブランドデザイン納品物 [リンク] から、
顧客側で自走できるための運用ガイド（Markdown）を起案。
構成:

1. ブランドの核（コア価値 / トーン）— 1ページ
2. やってよいこと / やってはいけないこと — 対比表
3. ロゴ使用ルール — 図解付き
4. テンプレート集 — 使い方 + アクセス先
5. 困ったときの判定フロー — 「これってブランドに合う？」の5問
6. 改変リクエスト窓口 — 連絡先と対応期間
```

### 引き継ぎMTGアジェンダ

```
引き渡し相手 [エンジニア / 顧客デザイナー / etc.] 向けに、
30分構成の引き継ぎMTGアジェンダを起案:

- 5分: PJ概要とデザインインテント（なぜこの形になったか）
- 10分: 納品物の構造ツアー（Figma / ガイドライン / テンプレート）
- 5分: 実装注意点（特にミスが起きやすいポイント）
- 5分: QAチェックリスト（受け取り側が自分で確認する項目）
- 5分: 質問窓口・SLAの合意

各セクションで「相手が必ず話す」時間を組み込む。一方的な説明にしない。
```

### デザイン→コード ギャップ予測

```
Figmaファイル [リンク] と想定実装環境 [React / Next.js / etc.] から、
デザイン → コードのギャップが生じやすいポイントを5つリストアップ。
各ポイントに「事前に伝えておくべきこと」を1行ずつ。

特に：
- フォント描画差（macOS / Windows / Web）
- アニメーション再現性
- 余白の解釈（Figmaの padding と CSS の margin-collapsing）
- 色のレンダリング（sRGB / DisplayP3）
- レスポンシブの中間状態
```

## 引き渡し後の付き添い

- 引き渡し後 **2週間** は、軽い質問に都度対応（無償・15分以内のもの）
- 2週間超えの追加相談は **別契約として案内**（曖昧にしない）
- 引き渡し後 **30日** で `internal-marketing/case-study.md` の事例化打診へ

## 注意

- **資料を渡したら終わり、ではない**。引き渡しMTGの **対話** が引き継ぎの本体
- 引き渡し相手が自分より知識が浅くても、敬意をもって接する（一緒に磨いた成果を、次の人が育てる）
- 顧客側が「内製化」を望んでいる場合、過剰な手厚さは逆に依存を生む。**手放す勇気** も必要
