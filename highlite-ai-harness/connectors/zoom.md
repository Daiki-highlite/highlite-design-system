# Zoom — 録画・議事録の置き場と参照ルール

すべてのMTGを録画し、文字起こしから自動議事録を生成する。**人間が議事録を書く時間を減らす** が目的。

## 録画方針

### 録画対象

| MTG種別 | 録画 | 文字起こし | 議事録自動化 |
|---|---|---|---|
| Ore: 初回MTG / Discovery | 必須 | 必須 | 必須 |
| Ore: 提案MTG | 必須 | 必須 | 必須 |
| Lite: キックオフ | 必須 | 必須 | 必須 |
| Lite: 定例 | 必須 | 必須 | 必須 |
| Lite: クライアントレビュー | 必須 | 必須 | 必須 |
| 社内: 1on1 | 任意（本人合意必須） | 任意 | 任意 |
| 社内: 経営判断 | 必須 | 必須 | 任意 |
| 社内: ブレスト | 任意 | 任意 | — |

### 録画前のお作法

- 開始時に「録画します」と明示。顧客側にも同意を確認
- 議題を冒頭1分で口頭共有（後の文字起こし要約の精度向上）
- 機密情報を扱う場合は事前に「このパートは録画停止」を宣言

## 保存先

```
Zoom Cloud（自動保存）
└─ 自動でDriveに同期: /Highlite Drive/02_Lite/[PJ名]/02_Cadence/recordings/
```

ファイル名規則:
```
YYYYMMDD_[PJ名]_[MTG種別]_v01.mp4
YYYYMMDD_[PJ名]_[MTG種別]_v01.txt   # 文字起こし
YYYYMMDD_[PJ名]_[MTG種別]_v01_minutes.md   # AI生成議事録
```

例:
- `20260427_acme_kickoff_v01.mp4`
- `20260504_acme_weekly04_v01_minutes.md`

## 議事録自動化フロー

```
1. Zoomで録画 → クラウドに自動保存
2. 文字起こしを待つ（クラウド版は30分〜2時間）
3. 完了時に AI Bot が以下を実行:
   a. 文字起こしを取得
   b. 議事録テンプレに沿って要約生成
   c. Drive 該当フォルダに `_minutes.md` で保存
   d. `#lite-lapidaristデリバリー-cc` に通知
```

## 1on1の取り扱い（特例）

- **録画は本人の事前同意があるときのみ**
- 文字起こし後、**メモ化したらオリジナル文字起こしは削除**
- 議事録は本人 + DH のみアクセス可
- AIに渡すコンテキストは「処理後忘却」を指示

## 注意

- **顧客同意なき録画は禁止**
- **文字起こしの誤字・誤聞き** を前提に運用。重要決定は議事録レビュー必須
- **生のZoom録画リンクの長期保存** は避ける（容量・コスト）。3ヶ月以上前は文字起こし + 議事録のみ残し、録画は削除
- **1on1録画は90日以内に必ず削除**（人事情報の長期保存リスク）
- **議事録テンプレを変えない** — 検索性とAI処理の安定性のため
