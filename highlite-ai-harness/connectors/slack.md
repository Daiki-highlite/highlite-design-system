# Slack — チャンネル構造と運用ルール

小規模組織向けに **4チャンネルへ集約**。新規作成しないことを規律にする。

## 4つの cc チャンネル

| チャンネル | 集約役割 | 主な投稿者 |
|---|---|---|
| `#ore-リード・アサイン情報-cc` | Ore全フェーズ + 担当アサイン | BizDev / DH / AI Bot |
| `#lite-lapidaristデリバリー-cc` | Lite運用全般（ブランドデザイン含む） | Delivery Lead / メンバー / AI Bot |
| `#lite-請求・入金-cc` | 請求書発行 / 入金管理 | DH / AI/Ops / AI Bot |
| `#cry-ナーチャリング・イベント-cc` | Crysta全フェーズ + 月次索敵レポート | DH / Marketing / AI Bot |

`-cc` サフィックス = "Cowork Channel" を意味する社内規約。

## 投稿ルール

### 必須要素

各 cc チャンネルへの投稿は、以下を必ず含める:

```
:emoji: [フェーズ]-[ステップ番号] [ステップ名]

▼ [カテゴリ]
[内容]

▼ [カテゴリ]
[内容]

詳細: [リンク]
```

各サブスキルの末尾「Push文テンプレ」を参照。

### 通知強度

| 強度 | 使う場面 | チャンネル / 表記 |
|---|---|---|
| 即時 + `@here` | SLA超過 / 高優先度ニーズ / 入金期日超過 / 検収完了 | 該当 cc に直接 |
| 即時（メンションなし） | 通常のステップ完了 / 進捗共有 | 該当 cc |
| デイリー | （廃止） | — |
| ウィークリー | PJ進捗サマリ（金17時） | `#lite-lapidaristデリバリー-cc` |
| マンスリー | KPI / 索敵レポート / 入金サマリ（月初9時） | 各 cc |
| DM | 個別リマインダ / 1on1関連 | 個別 |

## メッセージ運用

### スレッド使い方

- **チャンネル投稿**: 1トピック1投稿。続報・議論はスレッドへ
- **スレッド**: 関連する応答・質問はすべてスレッドに収める（チャンネルを汚さない）
- **ブロードキャスト**: スレッドから本文へ流すのは「全員に再告知が必要な場合」のみ

### ピン留めの基準

各 cc に常時ピン留めしておくもの:
- チャンネルの目的（1行）
- 関連するハーネス内SKILLへのリンク
- 緊急時連絡フロー
- 主要なテンプレ（議事録 / アジェンダ）

### リアクション運用

- :white_check_mark: = 確認した
- :eyes: = 見ている
- :raised_hands: = 担当する
- :thinking_face: = 議論したい
- 絵文字は感情より状態表示に使う

## 検索のための工夫

- 投稿冒頭の `:emoji: [フェーズ]-[ステップ番号]` 形式で検索性を担保
- 例: `[Ore]-02` で全インバウンド着弾通知が引ける
- 顧客名は伏字 + ID併記で検索ヒットさせる: `[Acme] / O-2026-007`

## AI Bot の振る舞い

ハーネス側のスキルから自動投稿される際は:

```
[Bot user名]: Highlite Harness Bot
[投稿前必須]: ../skills/_shared/brand-voice.md のチェック項目を通す
[禁止]: 過剰な絵文字 / フレンドリー過ぎるトーン / メンションの乱発
[Push文]: 各サブスキルの「Push文テンプレ」に厳密に従う
```

## 新規チャンネル作成の判断

**原則: 作らない**。既存4本に押し込めないか先に検討。

例外的に検討してよいケース:
- 顧客固有の機密性が極めて高く、Highlite社員以外に見せてはいけない一時案件
- 単発イベント（共催ウェビナー）の運営期間限定（終了後アーカイブ）

新規作成する場合の必須ステップ:
1. DH の承認
2. 命名規則 `[フェーズ]-[内容]-cc` または `event-[名称]-cc`
3. アーカイブ条件を作成時に決める
4. 本ドキュメントの「4つの cc チャンネル」表に追記しない（一時的扱い）

## アンチパターン

- 新規チャンネル乱立 → 集約に戻す
- ピン留め肥大化 → 古いものを定期的に外す
- リアクションだけで会話を完結させる → スレッドで対話する
- ボット投稿を読まない → 通知設計を見直す（おそらく多すぎる）
- 個別案件専用チャンネルの濫用 → 既存4本のスレッドに押し込む
